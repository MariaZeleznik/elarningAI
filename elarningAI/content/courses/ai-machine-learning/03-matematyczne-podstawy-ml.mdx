## Wektory i macierze - dlaczego to ważne

Każdy przykład w ML to wektor. Jeśli masz dane o 150 kwiatach z 4 cechami, masz macierz 150×4. Trening modelu to operacje na tych macierzach. Tyle.

```python
import numpy as np

# Jeden kwiat to wektor: [długość działki, szerokość działki, długość płatka, szerokość płatka]
kwiat = np.array([5.1, 3.5, 1.4, 0.2])

# Cały dataset to macierz
dataset = np.array([
    [5.1, 3.5, 1.4, 0.2],
    [4.9, 3.0, 1.4, 0.2],
    [6.3, 3.3, 6.0, 2.5],
])
print(f"Kształt datasetu: {dataset.shape}")  # (3, 4)
```

Iloczyn skalarny (dot product) to serce regresji liniowej. Kiedy model liczy predykcję, robi dokładnie to: `ŷ = X @ wagi + bias`. Jedna linia kodu, miliony operacji równolegle na GPU.

## Gradient - co to właściwie oznacza

Gradient to kierunek najszybszego wzrostu funkcji. Gradient descent idzie w PRZECIWNYM kierunku, bo chcemy minimalizować błąd, nie maximizować.

```python
# Prosta ilustracja: szukamy minimum funkcji f(x) = x^2
def f(x): return x**2
def gradient_f(x): return 2*x  # pochodna

x = 5.0  # start daleko od minimum (x=0)
lr = 0.1

for krok in range(20):
    grad = gradient_f(x)
    x = x - lr * grad  # idź przeciwnie do gradientu
    if krok % 5 == 0:
        print(f"Krok {krok}: x={x:.4f}, f(x)={f(x):.4f}")
```

Po 20 krokach x będzie blisko 0. To samo robi gradient descent w sieciach neuronowych - ale z milionami parametrów jednocześnie.

## Prawdopodobieństwo - minimalne co musisz wiedzieć

Modele klasyfikacji zwracają prawdopodobieństwa. Softmax zamienia logity na prawdopodobieństwa sumujące się do 1.

```python
import numpy as np

def softmax(logity):
    e = np.exp(logity - np.max(logity))  # stabilność numeryczna
    return e / e.sum()

logity = np.array([2.0, 1.0, 0.1])
prob = softmax(logity)
print(prob)  # [0.659, 0.242, 0.099] → sumuje do 1.0
```

Klasy z najwyższym prawdopodobieństwem = predykcja modelu.
