Gdy pierwsza sieć neuronowa widziała tekst, widziała go jak worek słów - kolejność nie miała znaczenia. Potem ktoś zapytał: a co jeśli "pies ugryzł człowieka" i "człowiek ugryzł psa" to różne rzeczy? Tak powstały sieci rekurencyjne.

## Dlaczego MLP nie radzi sobie z sekwencjami

MLP przetwarza każdy przykład niezależnie - nie pamięta poprzednich wejść. Ale przy sekwencjach kontekst jest wszystkim:

- "Jutro będzie **ładna** pogoda" (ładna = przymiotnik)
- "Ona ma **ładną** twarz" (też przymiotnik, ale inny kontekst)
- Następne słowo po "Warszawa to stolica..." - wymaga wiedzy o Polsce

## Recurrent Neural Network (RNN) - intuicja

RNN ma **ukryty stan** (hidden state) przenoszony między krokami sekwencji:

```
h₀ → [RNN] → h₁ → [RNN] → h₂ → [RNN] → h₃ → ... → predykcja
       ↑             ↑             ↑
      x₁            x₂            x₃    (kolejne elementy sekwencji)
```

```python
import torch
import torch.nn as nn

# Prosta RNN: wejście 10, hidden state 32, 1 warstwa
rnn = nn.RNN(input_size=10, hidden_size=32, batch_first=True)

# batch=2, sekwencja 5 kroków, 10 cech na krok
x = torch.randn(2, 5, 10)
output, h_n = rnn(x)
print(f"Output: {output.shape}")  # (2, 5, 32) - hidden state dla każdego kroku
print(f"h_n: {h_n.shape}")        # (1, 2, 32) - ostatni hidden state
```

**Problem z prostym RNN:** vanishing gradient. Przy długich sekwencjach (100+ kroków) gradient "zanika" podczas backpropagation - sieć zapomina co było na początku sekwencji.

## LSTM - Long Short-Term Memory

LSTM rozwiązuje problem znikającego gradientu przez mechanizm **bramek** (gates):

```
       Forget Gate    Input Gate     Output Gate
           ↓               ↓              ↓
h_{t-1} → [f] →       [i] ×          [o] ×
               ↘         ↙    C_t    tanh(C_t) → h_t
                cell state (pamięć długoterminowa)
```

Intuicyjnie: LSTM może "wybrać" co zapamiętać, co zapomnieć i co przekazać dalej.

```python
import torch
import torch.nn as nn

lstm = nn.LSTM(
    input_size=50,    # wymiar embeddingów słów
    hidden_size=128,  # rozmiar ukrytego stanu
    num_layers=2,     # warstwy LSTM
    batch_first=True,
    dropout=0.3,      # dropout między warstwami
    bidirectional=True,  # czyta sekwencję w obu kierunkach
)

# Wejście: batch=8, sekwencja 100 słów, embedding 50
x = torch.randn(8, 100, 50)
output, (h_n, c_n) = lstm(x)

print(f"Output: {output.shape}")  # (8, 100, 256) - 128*2 bo bidirectional
print(f"h_n: {h_n.shape}")        # (4, 8, 128) - 2 warstwy * 2 kierunki
```

## Praktyczny przykład: analiza sentymentu

```python
import torch
import torch.nn as nn
from torch.utils.data import Dataset, DataLoader
import numpy as np

class SentimentLSTM(nn.Module):
    def __init__(self, vocab_size, embed_dim, hidden_dim, output_dim):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, embed_dim, padding_idx=0)
        self.lstm = nn.LSTM(
            embed_dim, hidden_dim,
            num_layers=2, batch_first=True, dropout=0.3, bidirectional=True
        )
        self.fc = nn.Sequential(
            nn.Linear(hidden_dim * 2, 64),
            nn.ReLU(),
            nn.Dropout(0.3),
            nn.Linear(64, output_dim),
        )
    
    def forward(self, x):
        embedded = self.embedding(x)      # (batch, seq, embed_dim)
        output, (h_n, _) = self.lstm(embedded)
        # Połącz ostatni hidden state z obu kierunków
        final = torch.cat([h_n[-2], h_n[-1]], dim=1)
        return self.fc(final)

# Model dla 10 000 słów, embedding 128, hidden 256, 2 klasy (pozytywny/negatywny)
model = SentimentLSTM(
    vocab_size=10000,
    embed_dim=128,
    hidden_dim=256,
    output_dim=2,
)
print(f"Parametry: {sum(p.numel() for p in model.parameters()):,}")
```

## Szeregi czasowe - prognozowanie sprzedaży

```python
import torch
import torch.nn as nn
import numpy as np

# Symuluj dane sprzedaży (12 miesięcy → przewiduj miesiąc 13)
def create_sequences(data, seq_len):
    X, y = [], []
    for i in range(len(data) - seq_len):
        X.append(data[i:i+seq_len])
        y.append(data[i+seq_len])
    return np.array(X), np.array(y)

np.random.seed(42)
# Trendy + sezonowość + szum
t = np.arange(200)
sprzedaz = 100 + 0.5*t + 20*np.sin(2*np.pi*t/12) + np.random.normal(0, 5, 200)

X, y = create_sequences(sprzedaz, seq_len=12)
X = torch.FloatTensor(X).unsqueeze(-1)  # (samples, 12, 1)
y = torch.FloatTensor(y)

class ForecastLSTM(nn.Module):
    def __init__(self):
        super().__init__()
        self.lstm = nn.LSTM(1, 64, num_layers=2, batch_first=True, dropout=0.2)
        self.fc = nn.Linear(64, 1)
    
    def forward(self, x):
        out, _ = self.lstm(x)
        return self.fc(out[:, -1, :]).squeeze()  # tylko ostatni krok

model = ForecastLSTM()
optimizer = torch.optim.Adam(model.parameters(), lr=0.001)
criterion = nn.MSELoss()

for epoch in range(100):
    model.train()
    optimizer.zero_grad()
    pred = model(X)
    loss = criterion(pred, y)
    loss.backward()
    optimizer.step()
    if epoch % 20 == 0:
        print(f"Epoch {epoch}: MSE = {loss.item():.2f}")
```

## LSTM vs Transformery - co wybrać?

| Aspekt | LSTM | Transformer |
|--------|------|-------------|
| Sekwencje < 500 znaków | ✅ Dobry | ✅ Dobry |
| Długie sekwencje (>1000) | ❌ Problemy z pamięcią | ✅ Lepszy (attention) |
| Prędkość treningu | Sekwencyjna, wolna | Równoległa, szybsza |
| Szeregi czasowe | ✅ Naturalny | ✅ (z position encoding) |
| NLP (stan sztuki 2024) | ❌ Zdominowany | ✅ GPT, BERT, T5... |

W praktyce (2025): przy NLP zawsze transformer. LSTM nadal warto znać dla małych datasetów i embedded devices.
