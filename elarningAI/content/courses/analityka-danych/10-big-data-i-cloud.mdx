## Czym różni się Big Data od "normalnych danych"

Big Data to nie tylko "dużo danych". To dane które nie mieszczą się w RAM jednej maszyny, nie działają dobrze w tradycyjnym SQL, lub wymagają przetwarzania w czasie rzeczywistym.

Tradycyjne limity:
- Excel: < 1M wierszy
- Pandas w RAM: < 10-50M wierszy (zależnie od kolumn)
- PostgreSQL na jednej maszynie: miliardy wierszy, ale powoli

Cloud Data Warehouse (BigQuery, Redshift, Snowflake):
- Dziesiątki miliardów wierszy
- Kolumnowe przechowywanie (szybkie skanowanie kolumn)
- Masowe przetwarzanie równoległe (tysiące "slotów" CPU)
- Pay-per-query (płacisz za skany, nie za czas działania serwera)

## SQL w BigQuery - rozszerzenia i różnice

BigQuery używa Standard SQL z kilkoma dodatkami:

```sql
-- 1. Partycjonowanie i grupowanie po czasie
SELECT
  DATE_TRUNC(created_at, MONTH) AS miesiac,
  platform,
  COUNT(DISTINCT user_id) AS unikalni_uzytkownicy,
  COUNT(*) AS sesje,
  SUM(revenue_usd) AS przychod
FROM `analytics.sessions`
WHERE
  created_at BETWEEN '2024-01-01' AND '2024-12-31'
  AND platform IN ('web', 'mobile_ios', 'mobile_android')
GROUP BY 1, 2
ORDER BY miesiac DESC, przychod DESC

-- 2. UNNEST - rozwinięcie tablicy JSON
SELECT
  user_id,
  item.product_id,
  item.quantity,
  item.price
FROM `ecommerce.orders`,
UNNEST(items) AS item  -- items to kolumna ARRAY<STRUCT<...>>
WHERE order_date = '2024-03-15'

-- 3. STRUCT i zagnieżdżone dane
SELECT
  user_id,
  device.browser,
  device.os,
  geo.country,
  geo.city
FROM `analytics.events`
LIMIT 100

-- 4. Funkcje analityczne (window functions) - BigQuery obsługuje pełen standard SQL
WITH ranked AS (
  SELECT
    user_id,
    event_name,
    event_timestamp,
    ROW_NUMBER() OVER(PARTITION BY user_id ORDER BY event_timestamp) AS event_order,
    LAG(event_timestamp) OVER(PARTITION BY user_id ORDER BY event_timestamp) AS prev_event_ts,
    TIMESTAMP_DIFF(event_timestamp, LAG(event_timestamp) OVER(PARTITION BY user_id ORDER BY event_timestamp), SECOND) AS seconds_since_prev
  FROM `analytics.events`
  WHERE DATE(event_timestamp) = '2024-03-15'
)
SELECT
  event_name,
  ROUND(AVG(seconds_since_prev), 1) AS avg_seconds_to_reach,
  COUNT(*) AS occurrences
FROM ranked
WHERE event_order > 1
GROUP BY event_name
ORDER BY avg_seconds_to_reach
```

## Funnel Analysis w BigQuery

Analiza lejka sprzedażowego - ile użytkowników przeszło każdy krok:

```sql
WITH events AS (
  SELECT
    user_id,
    event_name,
    event_timestamp
  FROM `analytics.events`
  WHERE DATE(event_timestamp) BETWEEN '2024-03-01' AND '2024-03-31'
    AND event_name IN ('page_view', 'product_view', 'add_to_cart', 'checkout_start', 'purchase')
),
funnel AS (
  SELECT
    user_id,
    MAX(CASE WHEN event_name = 'page_view' THEN 1 ELSE 0 END) AS visited,
    MAX(CASE WHEN event_name = 'product_view' THEN 1 ELSE 0 END) AS viewed_product,
    MAX(CASE WHEN event_name = 'add_to_cart' THEN 1 ELSE 0 END) AS added_to_cart,
    MAX(CASE WHEN event_name = 'checkout_start' THEN 1 ELSE 0 END) AS started_checkout,
    MAX(CASE WHEN event_name = 'purchase' THEN 1 ELSE 0 END) AS purchased
  FROM events
  GROUP BY user_id
)
SELECT
  SUM(visited)          AS odwiedziny,
  SUM(viewed_product)   AS obejrzal_produkt,
  SUM(added_to_cart)    AS dodal_do_koszyka,
  SUM(started_checkout) AS rozpoczal_checkout,
  SUM(purchased)        AS kupil,
  ROUND(SUM(purchased) * 100.0 / SUM(visited), 2) AS conversion_rate_pct
FROM funnel
```

## dbt - transformacje danych jako kod

dbt (data build tool) to standard dla transformacji w data warehouse:

```sql
-- models/marts/fct_orders.sql
-- dbt kompiluje to do pełnego SQL i tworzy tabelę/widok w BigQuery

WITH source AS (
    SELECT * FROM {{ ref('stg_orders') }}  -- referencja do innego modelu dbt
),

z_klientami AS (
    SELECT
        o.*,
        k.customer_segment,
        k.country_code
    FROM source o
    LEFT JOIN {{ ref('dim_customers') }} k ON o.customer_id = k.customer_id
),

finalne AS (
    SELECT
        order_id,
        customer_id,
        customer_segment,
        country_code,
        order_date,
        SUM(quantity * unit_price) AS total_value,
        COUNT(DISTINCT product_id) AS unique_products
    FROM z_klientami
    GROUP BY 1, 2, 3, 4, 5
)

SELECT * FROM finalne
```

## Apache Spark - przetwarzanie Big Data po stronie kodu

```python
from pyspark.sql import SparkSession
from pyspark.sql.functions import col, sum, count, avg, date_trunc

spark = SparkSession.builder.appName("BigDataAnalysis").getOrCreate()

# Wczytaj dane (S3, HDFS, GCS lub lokalny plik)
df = spark.read.parquet("gs://my-bucket/events/2024/")

# Zapytania identyczne z pandas API, ale na klastrze
result = (
    df
    .filter(col("event_name") == "purchase")
    .withColumn("month", date_trunc("month", col("event_timestamp")))
    .groupBy("month", "platform")
    .agg(
        count("*").alias("purchases"),
        sum("revenue").alias("total_revenue"),
        avg("revenue").alias("avg_order_value"),
    )
    .orderBy("month", ascending=False)
)

result.show(20)
result.write.parquet("gs://my-bucket/results/monthly_purchases/")
```

| Rozmiar danych | Narzędzie | Koszt |
|---------------|-----------|-------|

| > 1B, streaming | Spark + Kafka | Drogi, ale konieczny |
